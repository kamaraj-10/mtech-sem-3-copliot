BITS Pilani, Deemed to be University under Section 3 of UGC Act, 1956The ReActFrameworkReasoning + Acting: Foundation of Agentic Systems
ReAct (Yao et al., 2023)Combines reasoning traces and task-speciﬁc ac,ons in an interleaved manner, allowing LLMs to solve complex tasks through explicit reasoning steps. The framework creates a loop where the model thinks, acts, observes, and con,nues reasoning.
Simple Example: Finding Country CapitalQues%onWhat is the capital of the country where Paris is located?ThoughtI need to first determine which country Paris is in.Actionsearch("Paris location")Observa%onParis is the capital of France.AnswerParis

1. Iteration LimitsProblem: Agent gets stuck in reasoning loopSolution: Set max iterations (typically 10). If limit reached, log history, return error to user, alert monitoring system.
2. Repetition DetectionProblem: Agent calls the same failed tool repeatedlySolu3on: Track ac,on signatures (tool name + args hash). If same ac,on appears 3×in history → force comple,on or try alterna,ve.
3. Tool Failure TrackingProblem: Problema,c tool causes repeated failuresSolu3on: Track failures per tool using defaultdict(int). Ater 3 consecu,ve failures, remove from available tools for this session.
4. Timeout ProtecOonProblem: Agent takes too long, tying up resourcesSolution: Implement 60-second alarm using signal module (Unix) or threading.Timer (cross-platform). Raise TimeoutError when exceeded.
5. Input ValidationProblem: Invalid function arguments cause crashesSolution: Use Pydantic models for all function parameters. Validate before execution. If validation fails, return structured error to model for retry.
6. Graceful DegradationProblem: Tool unavailable but task s,ll needs comple,onSolu3on: When tool fails, provide alterna,ve sugges,ons. Maintain fallback tools or guide model to alterna,ve approaches.
