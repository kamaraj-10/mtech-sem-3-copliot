The RLHF Architecture –A  Two model Dance
The Cyclical Training ProcessThe two models refine each other in a loop:•Feedback Signal:The Reward Model provides a scoring signal that tells the Policy Model which of its generated responses are better.•Model Optimization:The Policy Model uses this feedback (often via an algorithm like PPO) to update its internal parameters.•Data Refinement:As the Policy Model generates new, increasingly sophisticated data, that data can be used to further train and sharpen the Reward Model's accuracy.

BITS Pilani, Deemed to be University under Section 3 of UGC Act, 1956Adapter-Based Methods  (Houlsbyet al., 2019)•Vanilla fine-tuning can be seen as adding an extra layer on top of the Transformer•Adapter modules perform more general architectural modifications: injecting new layers/modules into the original network•During training, original network weights are untouched –only adapter weights update•Two small sub-layers: down-projection → non-linearity → up-projection•Near-identity initialisation so training stability is preserved•Only adapter layers + final classification layer update during training•Achieves GLUE performance within 0.4% of full fine-tuning using <2% of trainable parameters

BITS Pilani, Deemed to be University under Section 3 of UGC Act, 1956LoRA: Low-Rank Adaptation  (Hu et al., 2021)
Key InsightNeural networks contain many dense layers performing matrix multiplication. Inspired by the intrinsic dimension hypothesis, the weight updates ΔW during fine-tuning have low intrinsic rank.
Pre-trained matrix:W₀ ∈ℝ^(d×k) —frozen during training
Updated matrix:W₀ + ΔW = W₀ + BA
Low-rank matrices:A ∈ℝ^(r×k),  B ∈ℝ^(d×r),  where r ≪min(d,k)
Hidden state:h = W₀x  →  h = W₀x + ΔWx = W₀x + BAx
Rank r:Typically 2–64; r=4 or r=8 already gives strong resultsHu, Edward J., et al. "LoRA: Low-Rank Adaptation of Large Language Models." ICLR 2022.


--- Page 49 ---
BITS Pilani, Deemed to be University under Section 3 of UGC Act, 1956Applying LoRAto Transformers•In principle, LoRA can be applied to any weight matrices in deep learning•In practice, the focus is on applying LoRA to attention matrices in Transformers: Wq, Wk, Wv, Wo•Rank r ranges from 2 to 64 —typically r = 4 or r = 8 gives strong results•GPU VRAM requirement drops from 1.2 TB → >350 GB for large models•Checkpoint storage: 350 GB (full) → 35 MB (LoRA adapter) —10,000×smaller
Which Matrices? (Ablation —Hu et al., 2021)Applying LoRA to both Wq and Wv (attention query + value matrices) gives the best trade-off between parameter count and accuracy.r = 4 already gives strong results; r = 8 covers most meaningful subspaces for most tasks.

BITS Pilani, Deemed to be University under Section 3 of UGC Act, 1956Quantization Basics
Motivation: 32-bit floating-point computations are memory-intensive. Quantization reduces the precision of numbers used in the model, dramatically cutting memory.
int8 tensorRange: [−127, 127]  (16 levels)
int4 tensorRange: [−8, 7]  (16 levels, signed)
float32Range: ~1.18×10⁻³⁸ to 3.4×10³⁸
Quantization / Dequantization FormulasQuantize:  X[int8] = round( (127 / absmax(X[FP32])) ×X[FP32] )Dequantize:  dequant(X[FP32], X[int8]) = X[int8] / (127 / absmax(X[FP32]))
⚠Quantization with outliers leads to larger quantization errors.

--- Page 54 ---
BITS Pilani, Deemed to be University under Section 3 of UGC Act, 19564-bit NormalFloat(NF4)  (Dettmers et al., 2023)•Neural network weights typically follow a zero-centered normal distribution N(0, σ²)•NF4 is an information-theoretically optimal quantization type for normally distributed data•Normalization: transform all weights to a single fixed distribution by scaling with σ and setting range to [−1, 1] with mean 0 and standard deviation 1•Quantize the normalized weights to 4 bits —16 levels evenly spaced within the normalized values•NF4 yields better empirical results than both 4-bit integers and 4-bit floats•Small dequantization error: for example, 0.1832 rounds to 0.2 → error ≈ 0.0168
NF4 (NormalFloat)Information-theoretically optimal for normal distributions. Used in QLoRA.
int4 (Integer)Uniformly spaced integers. Not optimal when weight distribution has thin tails.
fp4 (Float)4-bit floats —fewer levels near zero where weights cluster.Dettmers, Tim, et al. "QLoRA: Efficient Finetuning of Quantized LLMs." NeurIPS 2023. https://doi.org/10.48550/arXiv.2305.14314

BITS Pilani, Deemed to be University under Section 3 of UGC Act, 1956QLoRA: Quantization + Low-Rank Adapters  (Dettmers et al., 2023)
QLoRA combines 4-bit NF4 quantization with Low-Rank Adapters to enable efficient fine-tuning of large language models on a single consumer GPU.
1. 4-bit NormalFloatPre-trained model weights quantized to NF4 and frozen. Significantly reduces memory usage at load time.
2. Double QuantizationQuantize the quantization constants themselves, saving ~0.37 bits/parameter (≈3 GB for a 65B model).
3. Paged OptimizersNVIDIA unified memory to avoid gradient checkpointing spikes when processing long-sequence mini-batches.
Memory reduction: Full FT of 65B model requires >780 GB GPU memory.  QLoRA trains same model in <48 GBwithout degrading performance vs. 16-bit fully fine-tuned baseline.

BITS Pilani, Deemed to be University under Section 3 of UGC Act, 1956QLoRAArchitecture & Training Flow
1Load Base ModelLoad pre-trained LLM (e.g. LLaMA 70B) in 4-bit NF4 format. Weights are frozen —no gradient flows through them.
2Add LoRA AdaptersAttach small trainable rank-r matrices A and B to each transformer layer (attention Wq, Wv). Adapters are in full 32-bit precision.
3Forward PassInput passes through quantized (frozen) base weights + full-precision LoRA adapters. Output = W₀x + BAx.
4Backward PassGradients flow only through LoRA matrices A and B. Quantized weights dequantized temporarily for computation; requantized after.
5Final ModelConsists of original 4-bit weights + trained adapter matrices in bfloat16/fp32. Can merge adapters into base model for zero-latency inference.Dettmers, Tim, et al. "QLoRA: Efficient Finetuning of Quantized LLMs." NeurIPS 2023.

Full Fine-TuningUpdates all weights. Highest accuracy but requires multi-GPU, per-task storage, and risks catastrophic forgetting.
Prompt / Prefix TuningSoft tokens added to input or every layer. Lester 2021 (input only), Li 2021 (every layer). Context window cost.
AdaptersInject trainable sub-layers (down→non-linear→up) into each transformer block. Near-FT performance with ~2% params. Houlsby 2019.
LoRAReparametrize ΔW = BA with rank r ≪d. Merge at inference → zero latency. 10 000×smaller checkpoints. Hu 2021.
QLoRA4-bit NF4 quantization + LoRA adapters in 32-bit. Double quantization. Paged optimizers. 65B model on <48 GB VRAM. Dettmers 2023.PEFT is now the default paradigm for adapting large language models to downstream tasks at low cost.

BITS Pilani, Deemed to be University under Section 3 of UGC Act, 1956KeyTakeaways
PEFT is now the default paradigmAdapting large pre-trained models without retraining all weights is the practical standard —enabling downstream tasks on consumer hardware.
Rank r is the core design choice in LoRAr = 4 or 8 covers most meaningful subspaces. Merging BA into W₀ at inference yields zero added latency —a key advantage over Adapters.
QLoRA democratises 65B+ model fine-tuningNF4 + double quantization + paged optimizers reduce a 65B fine-tune from >780 GB to <48 GB —single A100 territory.
Choose method by inference constraintPrompt/Prefix tuning adds context window cost; Adapters add latency; LoRA/QLoRA add neither at inference after merging.
Soft-prompt methods are model-scale dependentPrompt tuning matches full fine-tuning only at very large scale (Lester 2021). Prefix tuning extends coverage to every layer but consumes context budget.
Tiny adapters enable massive storage savingsOne shared base model + MB-sized adapter files per task replaces storing separate full-model copies, enabling practical multi-task deployment.The goal: match full fine-tuning accuracy at a fraction of the compute, memory, and storage cost.
