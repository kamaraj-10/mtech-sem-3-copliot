BITS Pilani, Deemed to be University under Section 3 of UGC Act, 1956Why Do We Need KV-Cache?Understanding Attention Computation in Autoregressive GenerationThe Inefficiency Problem: During text generation, transformers generate one token at a time. Each new token must attend to ALL previous tokens — recomputing K, V from scratch is O(n²) total work.Without KV-Cache (Naïve)Token 1: "The" → Compute Q₁, K₁, V₁Token 2: "cat" → Compute Q₁,K₁,V₁, Q₂,K₂,V₂  (recompute!)Token 3: "is" → Compute Q₁,K₁,V₁, Q₂,K₂,V₂ , Q3,K3,V3 Recompute all Q,K,V againToken n: Recompute everything for tokens 1 to n-1✗  Cost grows as O(n²) for n tokens!With KV-Cache (Optimised)Token 1: "The" → Compute K₁, V₁ → Cache themToken 2: "cat" → Reuse K₁,V₁; compute K₂,V₂ → CacheToken 3: "is" → Reuse K₁,K₂,V₁,V₂; compute K₃,V₃Token n: Reuse all cached K,V — compute only Kₙ,Vₙ✓  Only compute new K,V → O(n) total!Why Only Cache K and V?Query (Q): Changes with each new token position — must recompute.   Keys (K) & Values (V): Represent previous context — constant once computed. Cache them once, reuse forever.
